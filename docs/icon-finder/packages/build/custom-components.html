Building Iconify Icon Finder: custom components

This tutorial is part of Iconify Icon Finder build process tutorial.

Build process can swap files when building components. If you set option common.components.customDir in configuration, build script will look for files from that directory. Directory is relative to your UI package.

Example:

{
   "common": {
       "components": {
           "customDir": "src/components"
       }
   }
}

If you have a file src/components/ui/Container.svelte, build script will use it instead of @iconify/search-components/src/ui/Container.svelte to build @iconify/search-components/lib/ui/Container.svelte.

This applies to all files used by components: .svelte and .ts.

Examples

For examples of how it is done, look in Iconify for Figma plug-in code (see ui/figma of Icon Finder repository).
