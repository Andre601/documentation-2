Router in Iconify Icon Finder Core

This tutorial is part of Iconify Icon Finder Core tutorial.

Every view has a route associated with it. Route makes it easy to save and restore Icon Finder state, so visitor can continue browsing page from the previous session.

A route is a simple object of type PartialRoute describing the current view. It has a route type, set of parameters and optional parent route. For more details about route types and parameters, see routes documentation.

Routes are handled by Router instance. To access Router instance, use router property of IconFinderCore instance or router property of Registry instance:

const router = core.router;
const router = registry.router;

Router has the following methods and properties:

  • partialRoute, PartialRoute. Current route (only customised parameters). You can change route by changing value.
  • fullRoute, FullRoute. Current route (full object). You can change route by changing value.
  • defaultProvider, string. You can use this to set default API provider when navigating to home page.
  • home(). Navigates to home page.
  • action(). Run action on the currently visible view.
  • setCustomIcons(). Set icons list for a custom view. Works only if a custom view is currently visible or is being loaded. It is intended to be used to manipulate icons list while custom view is active.
  • getCustomIcons(). Same as setCustomIcons(), but returns current data instead of setting new data.

Functions and properties that are meant for internal use:

  • render(). Renders visible view, the same as the data sent to the render callback, so it is a bit redundant.
  • createChildView(). Creates new child view from route.
  • setParentView(). Navigate few levels up in parent views tree. Identical to action('parent', 1).

partialRoute and fullRoute

Properties partialRoute and fullRoute can access or change current route. See routes documentation for details.

What is the difference between two properties?

Property partialRoute contains copy of current route, without parameters that match default values.

Property fullRoute accesses current route variable directly. It has all parameters, including optional parameters. Be careful when using this property, it is a reference to internal variable, not a copy, so your code should never modify it and never assume it will not be modified. Use partialRoute instead if you need to a copy of route.

Setting either property to null is identical to calling home() function.

defaultProvider

Property defaultProvider is used to set default API provider. It affects only home() function.

home

Function home() is used to navigate to home route. Home route is set in configuration and is affected by defaultProvider.

By default, home route is:

{
   "type": "collections"
}

action

Runs an action on the currently visible view. It has 2 parameters:

  • action, string. Action name.
  • value. Value, type is specific to action.

For more information, see actions documentation.

setCustomIcons

Updates list of icons in a custom view. Custom view must be visible or pending or function will fail.

It has 2 parameters:

  • customType, string. Custom view type, such as "recent".
  • icons, IconsList. List of icons.

For more details about custom views, see custom views documentation.

getCustomIcons

Same as setCustomIcons() above, but has only customType parameter.

Returns list of icons as array of Icon objects.

render

Returns list of blocks for the current view, same as first parameter of the render callback.

This function is redundant because you already have all data available from render callback and is meant to be used internally when calling render callback.

For more details, check out IconFinderCore documentation, render callback documentation, blocks documentation.

createChildView

Creates a child view. It has 2 parameters:

  • route, PartialRoute. Route for child view.
  • parentLevels, number. Optional number of levels to go up the parent routes tree before adding new child view. If set and is above 0, this would be identical to going up the parent views tree few levels up then adding a new child view.

setParentView

Navigates few levels up the parent routes tree. Optional parameter is the number of levels to go up, the default value is 1.
