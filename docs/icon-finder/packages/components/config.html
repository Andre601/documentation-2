Components Wrapper

Components configuration

This tutorial is part of Iconify Icon Finder components tutorial.

Components configuration is applied during compilation. It is stored in components property of configurator.json in projects.

See build process documentation for more information about configuration files and build process.

You can find the latest version of configuration options for components package and list of default values in file build/config.ts.

There are several sections of components configuration.

Language

language property points to the file that contains translation, relative to directory phrases of source directory (packages/components/src/).

Example:

{
   "components": {
       "language": "de"
   }
}

This will cause components package to use phrases/de.ts instead of phrases/en.ts.

Collections list

collectionsList property has options for the collections list. It changes how icon sets are rendered.

{
   "components": {
       "collectionsList": {
           "authorLink": false,
           "clickable": false
       }
   }
}

If you set clickable to true, an entire block will be clickable. This is useful if you want to display icon sets as small blocks.

Make sure theme supports various icon sets layouts.

Footer

Properties showFooter and footer change footer block.

Disabling footer

By setting showFooter to false, an entire footer is disabled and options in footer property become irrelevant. Footer components will not be bundled, reducing bundle size.

{
   "components": {
       "showFooter": false
   }
}

Use this option if you want to use custom footer. You can use callback from Wrapper instance to track selected icon and render your own footer.

Components

Components option allow you to swap various footer components. Different components render parts of footer in different ways.

Value for each option is a name of .svelte file. If component name is in lower case, it will be converted to "CamelCase".

Footer component

For example, footer.components.footer points to a container component for footer. Component should be in directory ui/footer/ of components source.

List of available components can change, at the moment of writing there are 3 options:

  • "full": uses ui/footer/Full.svelte. It renders big footer with large sample of the selected icon.
  • "simple": uses ui/footer/Simple.svelte. It renders compact footer with small sample of the selected icon.
  • "empty": uses ui/footer/Empty.svelte. It renders only footer buttons. No sample, no customisation options.
{
   "components": {
       "footer": {
           "components": {
               "footer": "simple"
           }
       }
   }
}

Example of configuration above could be confusing because of duplicate names:

  • components refers to components package configuration.
  • components.footer refers to footer configuration.
  • components.footer.components refers to list of components used in footer.
  • components.footer.components.footer refers to component used as footer wrapper.

Name component

Property footer.components.name points to a component used to display name of the selected icon. Component should be in directory ui/footer/parts/name/ of components source code.

List of available components can change, at the moment of writing there are 3 options:

  • "block": uses ui/footer/parts/name/Block.svelte. Renders selected icon name as big block, including mini sample.
  • "simple": uses ui/footer/parts/name/Simple.svelte. Shows sample separately from block.
  • "simple-editable": uses ui/footer/parts/name/SimpleEditable.svelte. Same as "simple", but selected icon name is an input box. Input is editable and if user's input matches an existing icon, it changes selection.

If icon name is displayed as input box, it will always include full icon name, such as line-md:bell.

If icon name is displayed as simple text and option footer.canShortenName is set to true, selected icon name will be shortened from line-md:bell to bell when viewing icon set, resulting in cleaner UI.

Customisations

footer.customisations property contains list of icon customisations. You can toggle color picker, icon dimensions, transformations and change components used for those elements.

Changing components works similar to footer components mentioned above. Components for various customisations are in directory ui/footer/parts/props/ in separate sub-directories for each type of customisation.

Buttons

footer.buttons property contains list of buttons used in footer.

Buttons will be displayed in exactly the same order as in configuration. In two examples below buttons are identical, but their order will be different:

{
   "components": {
       "footer": {
           "buttons": {
               "submit": {
                   "type": "primary",
                   "text": "Submit",
                   "icon": "line-md:confirm"
               },
               "cancel": {
                   "type": "secondary",
                   "text": "Cancel",
                   "icon": "line-md:close",
                   "always": true
               }
           }
       }
   }
}
{
   "components": {
       "footer": {
           "buttons": {
               "cancel": {
                   "type": "secondary",
                   "text": "Cancel",
                   "icon": "line-md:close",
                   "always": true
               },
               "submit": {
                   "type": "primary",
                   "text": "Submit",
                   "icon": "line-md:confirm"
               }
           }
       }
   }
}

Key in each object is used in Wrapper callback when button is clicked.

Value contains:

  • Button type: "primary", "secondary" or "destructive".
  • Button title. It is optional if translation for button is available in phrases in footerButtons object. Key in translation is the same as footer key, such as "submit" and "cancel" in examples above.
  • Icon is optional name of icon to show for button. You can add custom icons in theme's theme.json.
  • If optional property always is set to true, button will always be shown, even if there is no selected icon. This is used to display buttons such as close button that should always be visible.

If footer.buttons property is not set, components will render 2 buttons: "Submit" and "Cancel".

Other properties

There are more properties available.

See packages/components/build/config.ts for list of properties, their descriptions and default values.
