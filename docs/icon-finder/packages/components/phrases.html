Phrases in Iconify Icon Finder

Icon Finder UI is designed to be easy to customise. That includes the possibility of translation.

By default, Icon Finder UI uses English phrases. Phrases are written in TypeScript and are located in src/phrases/en.ts or components package.

Components package is located in packages/components/ of Icon Finder repository.

Creating a language pack

To create language pack, go to src/phrases/ of components package and copy en.ts. Filename must end with .ts.

Edit text in a new file.

Changing language pack

To change the default language pack, open configurator.json of UI package and change the value of property components.language:

{
   "components": {
       "language": "de"
   }
}

Language value is the name of your language pack file, without .ts extension.

After changing the language pack, rebuild UI.

Important: file configurator.json is not in components package, it is in package with Icon Finder UI. See build process documentation for details.

Variables

Some phrases contain variables.

For example, search.placeholder.collection has value "Search {name}".

In this case, "{name}" is a variable. It will be replaced by icon set name, such as "Material Design Icons", resulting in "Search Material Design Icons".

Variables should not be changed:

  • Do not translate variables. They are keywords, not words to translate.
  • Do not add new variables, it won't work.
  • Do not remove variables from phrases.

Special characters

In some phrases you'll see strange things like this: String.fromCharCode(0x00b0). Those are characters that cannot be reliably saved in a text file, such as degrees character.

Use basic set of characters in phrases. If you need to use a special character, use String.fromCharCode() function.

Rebuilding UI

This is specific to UI. Usually, you need to run npm run build for UI after changing configurator.json.

See build process documentation for details.
