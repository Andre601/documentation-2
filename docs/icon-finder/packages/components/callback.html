Components wrapper callback

This tutorial is part of Iconify Icon Finder components tutorial.

Callback must be provided as parameter to Wrapper constructor. It is used to notify your code whenever something changes in Icon Finder instance.

Callback is a function with only one parameter: event, IconFinderEvent.

Payload

Payload has type IconFinderEvent, which you can find in src/wrapper/events.ts of components package (to import type in your TypeScript code, use @iconify/search-components/lib/wrapper/events).

It is a simple object with multiple properties. Property type defines type of event, other properties are specific to that type.

Load

Load event is sent when Icon Finder has loaded.

Callback payload example:

{
   "type": "load"
}

Route

Route event is sent when current page has changed.

Event has property route, PartialRoute that contains new route.

Callback payload examples:

{
   "type": "route",
   "route": {
       "type": "collections"
   }
}
{
   "type": "route",
   "route": {
       "type": "collection",
       "params": {
           "prefix": "mdi",
           "page": 1
       },
       "parent": {
           "type": "collections"
       }
   }
}

Icon

Icon event is sent when new icon has been selected.

Event has property icon, Icon | null that contains new selected icon as object. If icon has been deselected, value is null.

Callback payload examples:

{
   "type": "icon",
   "icon": {
       "provider": "",
       "prefix": "mdi",
       "name": "account-lock"
   }
}
{
   "type": "icon",
   "icon": null
}

You can use function iconToString from Icon Finder Core to convert icon object to string:

import { Wrapper } from '@iconify/search-components/lib/wrapper';
import { IconFinderEvent } from '@iconify/search-components/lib/wrapper/events';
import { iconToString } from '@iconify/search-core';

// ...
const wrapper = new Wrapper({
   // ...
   callback: (event: IconFinderEvent) => {
       switch (event.type) {
           case 'icon':
               if (event.icon) {
                   console.log('New icon selected:', iconToString(event.icon));
               } else {
                   console.log('Icon has been deselected');
               }
               break;
       }
   },
   // ...
});
// ...

Usually user cannot deselect current icon. To deselect icon, use Wrapper's selectIcon(null) function as response to clicking button.

import { Wrapper } from '@iconify/search-components/lib/wrapper';
import { IconFinderEvent } from '@iconify/search-components/lib/wrapper/events';

// ...
const wrapper = new Wrapper({
   // ...
   callback: (event: IconFinderEvent) => {
       switch (event.type) {
           case 'button':
               switch (event.button) {
                   case 'reset':
                       // Deselect current icon
                       wrapper.selectIcon(null);
                       break;

                   // ...
               }
               break;
       }
   },
   // ...
});
// ...

Customisations

Customisations event it sent when icon customisations have changed. For example, when user has changed icon color or rotated icon.

Event has property customisations, PartialIconCustomisations that contains list of customisations.

Callback payload examples:

{
   "type": "customisations",
   "customisations": {
       "color": "#f20"
   }
}
{
   "type": "customisations",
   "customisations": {
       "hFlip": true,
       "color": "#f20",
       "height": "24"
   }
}

Button

Button event is sent when button in footer has been clicked. Event has 2 properties:

  • button, string. Key of button that has been clicked. It is the same key as in components.footer.buttons object in configurator.
  • state, IconFinderState. Current state.

Callback payload example:

{
   "type": "button",
   "button": "submit",
   "state": {
       "icon": {
           "provider": "",
           "prefix": "mdi",
           "name": "account-box-outline"
       },
       "route": {
           "type": "collection",
           "params": {
               "prefix": "mdi"
           },
           "parent": {
               "type": "collections"
           }
       },
       "config": {
           "components": {
               "list": true
           }
       },
       "customisations": {
           "color": "#f20"
       }
   }
}
