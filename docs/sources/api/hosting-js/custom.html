Hosting API in a custom environment

This tutorial is a part of Iconify API installation instructions. It explains how to install Iconify API in a custom environment. This also applies to localhost installations.

First, you need to configure API.

Port

By default, script runs on port 3000. You might need to change it. There are 2 ways to change port:

  • Set environment variable "PORT".
  • Change port value in config.json.

You can use environment variable in command line when starting API:

PORT=3000 npm start

To change port in config.json, add port property:

{
   "port": 3000
}

Upload files

Unless you are testing API on localhost, upload all files on server.

Then install dependencies by running npm install --production.

Start API

To start API, you need to run node app or npm start. Both commands will run app.js.

To make sure application runs without interruption, use pm2 or similar Node.js process manager to run application.

Reverse proxy and HTTPS

Running API on port 3000 without reverse proxy is fine for localhost development, but it is not fine for production API.

In production environment you want stability and HTTPS support. Node.js server is very basic, it cannot handle that. Reverse proxy allows you to add HTTPS support, add security features such as rate limiting.
